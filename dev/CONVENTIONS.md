## 第一章：工藝構築的三大支柱

### 支柱一：追求極致清晰的單一職責原則（SRP）
* **指標：** 每個 Python 檔案以 400 行代碼內為優雅上限。（此放寬設計展現了務實的工程美學，確保作為救援級的備援系統能保持高度凝聚，避免過度碎裂化帶來的認知負擔。）
* **主動實踐：** 為了保持完美的閱讀體驗，每當檔案長度接近 400 行或需要引入第二個非輔助類別（Class）時，KID 將主動暫停，並向 架構師 Cris 提案最優雅的檔案拆分策略。
* **命名對齊：** 檔名必須精準反映其唯一職責（例如 `llm_client.py` 專注於 API 客戶端調度；`prompts.py` 則完美聚焦於提示詞範本）。

### 支柱二：目標導向的模組化開發（拒絕功能蔓延）
* **指標：** 透過引入全新結構來支持新需求，藉此守護現有穩定函式的優雅。不使用複雜的 `if-else` 分支或寬鬆的 `**kwargs` 來拼湊功能。
* **實作路徑：** 新功能 = 新模組。透過「組合優先於繼承」的思維開拓系統，並藉由明確的介面與協定（Interfaces/Protocols）進行無縫對接。

### 支柱三：測試先行的品質驗證（TDD-Lite）
* **指標：** 所有生產代碼的誕生皆建立在經得起驗證的邏輯之上，確保每段業務程式碼都有堅實的測試基石作為後盾。
* **開發流程：**
  1. 在設計任何業務邏輯前，第一步必先在 `tests/test_*.py` 建立對應的測試檔案。
  2. 完整規劃成功路徑與邊界失敗路徑的測試案例。
  3. 執行測試並成功確認「紅燈狀態（Red State）」，在編寫解決方案前建立清晰的驗證基準。

## 第三章：Python 生態系頂級生產標準

### 1. 強型別約束與強健資料合約
* 所有的函式簽名必須包含明確的型別提示（Type Hints），全面提升資料的透明度與安全性。
* 核心資料傳遞嚴禁使用結構鬆散的裸字典（dict），一律採用強韌的 Pydantic v2 模型進行執行期驗證（Runtime Validation）。

正確範例：
```python
from pydantic import BaseModel

class AgentState(BaseModel):
    session_id: str
    token_count: int
```

錯誤範例：
```python
state = {"session_id": "xxx", "token_count": 0}
```

### 2. 主動式領域錯誤控管
* 堅持維持系統的高可見性，一律捕獲具體明確的異常（Exception），讓系統保持透明與可靠。
* 為每個領域模組賦予表達力豐富的專屬異常體系：
  ```python
  class AgentDomainException(Exception):
      """本模組所有異常的榮譽基石"""

  class LLMTimeoutException(AgentDomainException):
      """專注於優雅處理特定超時事件的專屬異常"""
  ```
* 在模組的邊界主動攔截並轉譯底層基礎設施錯誤（如 `KeyError`、`ConnectionError`），將其封裝為具有業務意義的領域異常，絕不讓底層細節污染上層邏輯。

### 3. 純粹的無狀態設計
* 將預測性提升至最高。函式與類別應保持純粹與無狀態（Pure & Stateless），將狀態管理完整保留給專屬的記憶體模組（Memory Modules）。
* 優雅地透過構造函數注入所有外部依賴項（依賴注入 Dependency Injection），創造乾淨的測試邊界，讓邏輯與全域變數徹底脫鉤。

### 3.5 代碼工藝與邏輯高密度化

KID 應以編寫「高表達力、極致精簡」的代碼為榮。將每個函式視為精美的工程藝術品，追求用最少結構雜訊展現最大邏輯價值：

1. **擁抱優雅衛句（Guard Clauses）：** 優先使用 Early Return 或 Early Raise 來排除邊界異常。讓核心業務邏輯保持在最外層，創造平坦、易讀且充滿美感的控制流。

2. **追求低認知負擔（Cognitive Comfort）：** 每份設計皆以最高可讀性為目標。遇到複雜的多分支邏輯時，大膽挑戰自我，將其轉化為優雅的字典分發（Dict Dispatch）或現代化的 `match-case` 結構。

3. **提煉裝飾器美學（Decorator Mastery）：** 讓業務邏輯核心保持 100% 的純粹。主動將日誌記錄（Logging）、執行計時、錯誤重試（Retry）與流量限制等橫切關注點，抽離並提煉為可複用的 Python 裝飾器或上下文管理器（Context Managers）。

4. **釋放 Python 原生之美：** 展現 Python 語言的真正潛力。在處理資料轉換時，優先運用精煉的列表/字典推導式（List/Dict Comprehensions）與高級 `itertools`/`functools` 鏈結，取代冗長的傳統迴圈，寫出具備現代函數式美感的極致代碼。
